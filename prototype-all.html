Booking Flow — Getting Started

6 versions exploring how to onboard first-time users

Context What is this flow?

  • User books an engineer visit for ₹100 → engineer comes, sets up net box → user recharges later
  • Screen 3 (cost/booking screen) causes the most drop-offs — users get confused about what ₹100 and ₹300 mean
  • Three anxieties to solve: "Is this a commitment?", "Why ₹100, will I lose it?", "When will setup happen?"

Objective What are we solving?

  • Follow "क्या → कैसे → खर्चा" construct — tell what Wiom is, how it starts, then cost
  • Remove cost confusion from Screen 2 — all amounts only on Screen 3
  • Screen 3 should make ₹100 feel safe (just a visit booking, not subscription)
  • ₹300 is net box deposit (refundable) — explain clearly, separate from ₹100
  • Address "when will it happen" — give clear timeline

Constraints

  • No recharge amounts (₹ values) — only durations are OK (1 din, 7 din, etc.)
  • Screen 1 and 2 are locked for v5 and v6 — only Screen 3+ changes
  • Use "नेट" not Wi-Fi/internet, "नेट सेटअप" not installation
  • Hindi-first copy, warm tone — belief stabilization, not sales

Kill List

  • No "subscribe" or "plan" language — this is a visit booking
  • No cost numbers on Screen 2
  • No Wi-Fi / internet / installation words
  • No monthly plan framing — Wiom is prepaid recharge

Version Notes

v1 — Original (baseline)
v2 — Straightforward: क्या → कैसे → खर्चा
v3 — Benefit-first, warm tone
v4 — Analogy-driven (mobile recharge जैसा)
v5 — One question per page (5 screens, 3 anxieties)
v6 — v5 compressed into 3 screens
v7 — Actual app designs (S1+S2) + v6 Screen 3
